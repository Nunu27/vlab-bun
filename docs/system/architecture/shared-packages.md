# Shared Packages

This page covers the packages consumed as `workspace:*` dependencies across the three apps, other than `@vlab/clab`, `@vlab/clab-monitor`, and `@vlab/evaluator`, which have their own pages ([clab.md](clab.md), [clab-monitor.md](clab-monitor.md), [evaluator.md](evaluator.md)).

## `packages/@vlab/shared` — Cross-app types, enums, schemas, utils

Pure TypeScript, no build step (`"module": "src/index.ts"`), depends on `@sinclair/typebox` + `@standard-schema/spec`. Consumed via subpath exports, not a single barrel:

- `@vlab/shared/enums` → `src/enums.ts`
- `@vlab/shared/types` → `src/types.ts`
- `@vlab/shared/schemas` and `@vlab/shared/schemas/*` → `src/schemas/index.ts` + per-entity files
- `@vlab/shared/utils` → `src/utils.ts`
- `@vlab/shared/standard-schema` → `src/standard-schema.ts`

**`src/enums.ts`** — canonical value/type pairs shared by manager, worker, web:

- `connectionTypeValues`/`ConnectionType` (`rdp|vnc|ssh|telnet`) — terminal proxy protocol.
- `deviceKindValues` — the full Containerlab `kind` catalog (Nokia, Arista, Juniper, Cisco, MikroTik, generic Linux/VM, etc.), though `DeviceKind` itself is typed as loose `string`.
- `degreeLevelValues`/`DegreeLevel` (`D3|LJ|D4|S2`) — student academic program levels (institution-specific).
- `roleValues`/`Role` (`student|instructor|admin`).
- `labTypeValues`/`LabType` (`user|device-test`).
- `nodeHealthValues`/`NodeHealth` (`healthy|unhealthy|starting|died|destroyed|null`) — explicitly commented as mirroring `@vlab/clab-monitor`'s `NodeHealth`; **also re-mirrored independently in `@vlab/grpc`'s `commands.ts` and `@vlab/ws`'s `lab-session.ts` router** — three copies of the same union that must be kept in sync by hand. If you add a health state, grep all three locations.

**`src/types.ts`** — `Session` type (`{ id, role, useCAS? }`) used for auth/session context.

**`src/utils.ts`** — generic helpers: `chunk`, `toKebabCase`, `toTitleCase`.

**`src/standard-schema.ts`** — `toStandardSchema()` adapts a TypeBox `TSchema` into a `StandardSchemaV1` validator, used so both `@vlab/grpc` and `@vlab/ws` waycast routers can validate TypeBox schemas through the standard-schema spec.

**`src/schemas/*`** (TypeBox, one file per domain entity) — `admin`, `auth`, `cas`, `common`, `department`, `device-category`, `device-template`, `instructor`, `lab`, `student`, `study-program`, `toast`, `topology-template`, `user`. Notable:

- `common.ts` — shared building blocks: `Nullable()`, `RequestWithId()`, `NonEmptyString()`, a `DateSchema` transform (string ⇄ Date), `DateRange`.
- `lab.ts` — the full lab-authoring domain model: `LabDeviceNodeSchema`/`LabGroupNodeSchema`/`LabNoteNodeSchema` (canvas nodes with x/y), `LabEdgeSchema`, `LabTopologySchema` (devices/groups/notes/edges keyed by UUID), `LabCheckConfigSchema`/`LabChecksMapSchema` (evaluation check bindings: `nodeId`, `checkId`, `params`, `weight`), and `LabRequestSchema` (full lab creation/update payload).

This package is the single source of truth for domain vocabulary; both `apps/manager` and `apps/web` import from it directly (per `AGENTS.md`: workspace imports, not relative paths).

## `packages/@vlab/grpc` — Manager↔Worker RPC layer contract

Full protocol walkthrough: [protocols/grpc-manager-worker.md](../protocols/grpc-manager-worker.md). In brief: wraps `nice-grpc` autogenerated bindings (from `proto/worker.proto`) with a `waycast` typed router (`src/commands.ts`) tunneled as opaque msgpack bytes inside the proto's single bidirectional stream. `src/codec.ts` (`msgpackCodec`) and `src/async-queue.ts` (`AsyncQueue<T>`, a backpressure-safe async-iterable push queue) are the supporting primitives. `src/index.ts` re-exports everything plus derives generic helper types (`GrpcRoutes`, `GrpcRpcPayloadOf`, `GrpcRpcResponseOf`, `GrpcRpcCallbacks`).

## `packages/@vlab/ws` — Manager↔Web real-time protocol contract

Full protocol walkthrough: [protocols/websocket-waycast.md](../protocols/websocket-waycast.md). Parallel design to `@vlab/grpc` but for the browser side (Socket.IO transport instead of gRPC). `src/types.ts` defines `WSMeta = { private?: Role[] }` used as waycast's generic route-metadata type for role-gating; `src/index.ts` composes the final `AppRouter` from four sub-routers (`admin.ts`, `device-template.ts`, `lab.ts`, `lab-session.ts`).

## `packages/@jawit/*` — Generic (non-vLab-specific) ecosystem utilities

| Package                     | Purpose                                                                                                                                                                                                                                                                                                                                                                                                                               |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`@jawit/common`**         | Standardized API response envelope: `success()`/`failure()` wrappers plus `responses.*` helpers (`notFound`, `deleted`, `created`, `updated`) for consistent `{ success, message, data? }` shapes across backend and frontend.                                                                                                                                                                                                        |
| **`@jawit/elysia-caching`** | HTTP caching plugin for Elysia.js: ETag/`Last-Modified` handling with automatic `304`, pluggable `CacheAdapter` interface (Redis/memory/etc.), per-route `cached` macro (`ttl`, `useETag`, `useLastModified`), and a request-scoped `cache` object for dynamic key building (`cache.set/addPrefix/addSuffix`).                                                                                                                        |
| **`@jawit/paginator`**      | Type-safe pagination/filtering/sorting for Drizzle ORM (Postgres) + TypeBox. `createPaginator(db, table, { usableColumns, searchableColumns })` auto-generates a validation schema (`page`, `perPage`, `search`, `filters[]` with typed ops like `between`, `sortBy`, `sortOrder`) and returns `PaginatedData<T>` (`items` + `pageInfo`). Depends on `drizzle-typebox`.                                                               |
| **`@jawit/query`**          | Bridges Elysia's Eden Treaty client with TanStack React Query: `treatyQuery(eden)` proxies an Eden client so every endpoint gains `.useQuery()`, `.useMutation()`, `.useInfiniteQuery()`, plus envelope-aware error handling (`onSuccessMessage`/`onErrorMessage`) and query-cache helpers (`invalidateQuery`, etc.) — the layer that gives `apps/web` end-to-end typed data-fetching hooks straight off the manager's Elysia routes. |
| **`@jawit/zustand-helper`** | Zustand ergonomics: `createSelectors()` (auto `.use.<field>()` selectors), `createModalStore()` (factory for boolean/data-carrying modal state + open/close actions), `createScopedStore()` (`/react` subpath — React-context-scoped store instances for component subtrees, e.g. a data table).                                                                                                                                      |

All five are consumed as internal workspace packages (`workspace:*`) by `apps/manager` and/or `apps/web`; none have package-specific build scripts (consumed directly from `src/`).

## `packages/external/mikro-routeros` — MikroTik RouterOS API client

A vendored/forked third-party package (`mikro-routeros`, plain JS with `index.d.ts` typings, CommonJS — kept under `packages/external/` rather than as a `@vlab/*`/`@jawit/*` scoped package since it isn't vLab-authored). A lightweight Node.js client for the **MikroTik RouterOS API** (TCP port 8728/8729):

- `RouterOSClient(host, port=8728, timeout=30000)` with `.connect()`, `.login(user, pass)`, `.runQuery(command, params)` (auto-detects `?`-prefixed query params vs `=`-prefixed action params), `.stream(command, params)` (EventEmitter-based for continuous commands like `/interface/monitor-traffic` or `/log/listen`), `.close()`.
- Underlies [`@vlab/evaluator`](evaluator.md)'s `mikrotik` handler's sources/listeners (e.g. `/routing/ospf/instance/listen`, `/log/listen`) and [`@vlab/clab-monitor`](clab-monitor.md#network-interface-monitoring-networking)'s `network/mikrotik_ros.ts`. Also used directly in `tests/evaluator-e2e` to seed/verify router state.
