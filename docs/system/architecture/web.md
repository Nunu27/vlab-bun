# Web Frontend (`apps/web`)

`@vlab/web` — React 19 + Vite 7 + TanStack Router (file-based) + TanStack Query v5 + Zustand v5 + Shadcn UI (`radix-vega` style) + Tailwind v4. Talks to [`apps/manager`](manager.md) over HTTP (Eden Treaty) and over WebSocket (Waycast/Socket.IO).

Related: [architecture/overview.md](overview.md) · [protocols/websocket-waycast.md](../protocols/websocket-waycast.md) · [architecture/shared-packages.md](shared-packages.md)

Route "module" convention: every route directory may contain a private `-module/` folder (TanStack Router ignores `-`-prefixed dirs for routing) holding that route's `components/`, `stores/`, `columns.tsx`, `types.ts` — i.e. colocated, route-scoped code instead of a global feature folder.

## Routing (TanStack Router, file-based, in `src/routes/`)

Router is configured in `src/lib/router.ts`; routes are generated into `src/routeTree.gen.ts` by the `@tanstack/router-plugin` Vite plugin (`autoCodeSplitting: true`). Pathless layout segments use `_` prefix (e.g. `_dashboard`, `_admin`) — they add a layout/guard without contributing a URL segment.

**Top-level:**

- **`__root.tsx`** — root route. In `beforeLoad` it hydrates `useAuthStore` (`actions.refresh()`) if the user is unknown, and reads a `toast` cookie set by the backend redirect flow to show a one-shot toast. Wraps everything in `NuqsAdapter` (URL state) + `TooltipProvider`, imports `@web/lib/ws` (bootstraps the socket singleton), and renders `<Outlet/>`. Also does client-side redirect-after-login/logout via `<Navigate/>`.
- **`login.tsx`** (`/login`) — Login form (email/password via `useAppForm` + typebox validator `LoginRequest`) plus a "Login with CAS" link (`/api/auth/cas`, backend-driven SSO redirect). Guarded by `guestRoute()`.
- **`_dashboard.tsx`** — shared authenticated shell: `protectedRoute()` guard, renders `SidebarProvider` + `AppSidebar` + header (breadcrumb, theme toggle, `NavUser`) + `<Outlet/>` + footer with app version.
  - `_dashboard/index.tsx` (`/`) — renders the role-specific dashboard page (student/instructor/admin) based on `useAuthStore` role.
  - **`_dashboard/_admin.tsx`** — `privateRoute(["admin"])` guard layout.
    - `master/department`, `master/study-program` — CRUD master data.
    - `lab-data/device-category`, `lab-data/device-template` (+ `create`, `$id.edit`) — device catalog administration; device-template has a multi-tab form (basic info, connection, cost, env vars, network interfaces, resources) and a "Test Connection" WS-RPC action.
    - `user/admin`, `user/instructor`, `user/student` — user management CRUD per role, plus shared "change password" modal.
    - `-module/components/admin-dashboard-page.tsx` — realtime worker/infra dashboard.
  - **`_dashboard/_instructor.tsx`** — `privateRoute(["instructor"])` guard layout.
    - `my-lab` (index/`create`) and `my-lab/$labId` (index/`edit`) — instructor's own labs: overview, topology editor (`topology-tab.tsx`, built on `shared/topology`), instructions editor (MDX + lab-checks), enrollments tab (realtime roster).
    - `topology-template` (index/`create`/`$templateId`) — reusable topology templates instructors can apply to labs.
    - `-module/components/instructor-dashboard-page.tsx` — stats dashboard (lab count, active sessions, students, avg score).
  - **`_dashboard/_student.tsx`** — `privateRoute(["student"])` guard layout.
    - `lab/browse.tsx` — catalog of labs available to enroll in.
    - `lab/enrolled.tsx` — labs the student is enrolled in, with session history.
    - `lab/$labId/index.tsx` — lab detail + "Start Lab Session" button.
    - `-module/components/student-dashboard-page.tsx` — enrolled labs + upcoming lab items.
- **`routes/lab/$labId/session/$labSessionId/`** — the live lab-session workspace (outside the `_dashboard` shell, full-screen):
  - `index.tsx` — main session page: topology canvas (`SessionTopology`), resizable sidebar with markdown instructions + lab checks, session header/timer/submit button, onboarding tour (`driver.js`-based), session-conflict/overridden modals (multi-tab/session takeover handling), all wired to realtime WS events.
  - `node/$nodeId.tsx` — opens a dedicated in-browser console/terminal to one lab device via Guacamole, reacting to `node:[id]:health` websocket data and closing itself if the parent session is taken over by another client.

**Route guards** (`src/lib/middlewares.ts`, all read `useAuthStore.getState()` directly, no React hooks needed since they run in `beforeLoad`):

- `guestRoute()` → redirect to `/` if already logged in.
- `protectedRoute()` → redirect to `/login` if not logged in.
- `privateRoute(roles: Role[])` → redirect to `/` if not logged in or role not in `roles`.

## State management (Zustand)

**Global:**

- **`src/stores/auth-store.ts`** — `useAuthStore` (via `@jawit/zustand-helper`'s `createSelectors`, giving `.use.<field>()` selector hooks and `.use.actions()`). Holds:
  - `user?: AuthUser | null` (`undefined` = not yet checked, `null` = confirmed logged out) — inferred straight from the `api.auth.me.get` treaty response type.
  - `redirectUrl?: string` — post-login/logout redirect target coming from the backend.
  - Actions: `refresh()` (checks `session` cookie via `cookieStore`, calls `api.auth.me.get`), `login(credentials)`, `logout()` (clears cookie + `queryClient.clear()`).
  - `src/lib/ws.ts` subscribes to this store to auto connect/disconnect the socket when `user` becomes truthy/falsy.

**Feature-scoped** (mostly modal-open state, not server data — server data lives in TanStack Query):

- Per-CRUD-entity `-module/stores/*-modal-store.ts` (department, study-program, device-category, device-template, admin/instructor/student user, lab, topology-template, user password).
- `_admin/-module/stores/metrics-history-store.ts` — rolling time-series buffer of worker CPU/mem/storage % for the admin dashboard's history chart, fed by websocket `admin:worker:metrics` events.
- `device-template/-module/stores/env-form-store.ts`, `test-device-store.ts` — device-template form/test-connection transient UI state.
- `lab-session/-module/store/lab-session-modal-store.ts` — sidebar-open / conflict / overridden modal state for the live session page.
- `mdx-plugins/lab-checks/stores/` — three stores: `lab-checks-editor-store` (authoring state), `lab-checks-modal-store` (insert/edit check modal), `lab-checks-session-store` (student-session runtime check-completion state, via React context `LabChecksSessionProvider`).

**`shared/topology/stores/`** — the most elaborate store: a **slice pattern** Zustand store built with `createScopedStore` (React-context-scoped, so multiple independent topology instances can exist), composed of `device-slice.ts`, `edge-slice.ts`, `group-slice.ts`, `node-slice.ts`, `note-slice.ts`, `view-slice.ts` (pan/zoom). `index.ts` combines them into `TopologyStore` (`TopologyStoreProvider` / `useTopologyStore`), holding devices/edges/groups/notes, template catalog, viewport, current interaction `mode` ("select"|"connect"|"note"), and an `isEditor` flag distinguishing the instructor's editable canvas from the read-only/interactive student session canvas. Uses `zustand/middleware`'s `subscribeWithSelector` and `immer`'s array/map support.

**`shared/guacamole/stores/guacamole-connection-store.ts`** — scoped store tracking the remote-display connection lifecycle (`connecting|connected|error|disconnected`, error message).

All stores follow `@jawit/zustand-helper` conventions: `Store<State, Actions>` shape with actions nested under `actions`, `createSelectors` for auto-generated per-field hooks, `createScopedStore` for provider-scoped (non-singleton) stores.

## Data fetching (TanStack Query) & API client

**`src/lib/api.ts`**:

```ts
const client = treaty<App>(window.location.origin); // Eden Treaty, typed against apps/manager's App
export default treatyQuery(client.api, {
	onErrorMessage: toast.error,
	onSuccessMessage: toast.success
});
```

`treatyQuery` (from workspace package `@jawit/query`) wraps every Eden Treaty endpoint with React Query hooks directly, e.g. `api.lab({labId}).get.useQuery()`, `api.lab.post.useMutation()`, `.usePagination()`, `.useInfiniteQuery()`, plus cache helpers `.queryOptions()`, `.setQueryData()`, `.invalidateQuery(queryClient)`, `.ensureQueryData(queryClient)`. Success/error toasts are wired centrally. This is fully end-to-end typed against the manager's Elysia route tree (`import type { App } from "@manager/server"` — the path alias `@manager/*` → `../manager/src/*` in `tsconfig.json` is a **source-level type import, not a runtime dependency**).

- **`src/lib/query.ts`** — the shared `QueryClient`: `staleTime: 0`, `gcTime: 5min`, `retry: false`, no refetch on window focus/reconnect (reasonable given most freshness comes from websockets instead of polling).
- Loader pattern: route `loader()`s call `api.x.get.ensureQueryData(queryClient)` / `queryClient.fetchQuery(...)` to preload before render; components then call the matching `.useSuspenseQuery()` (works with the router's `pendingComponent`).
- **Pagination/infinite-list helpers** (`src/hooks/pagination/`): `useApiPagination(endpoint, {params})` wraps `.post.usePagination()` and merges URL-synced page/sort/search/filter state with server args; `useApiInfiniteList(endpoint, {params})` wraps `.post.useInfiniteQuery()`.
- **Forms + mutations**: `useApiForm(endpoint, options)` binds a treaty mutation endpoint straight to a TanStack Form (`useAppForm`), calling `mutation.mutateAsync(value)` on submit — the standard pattern for all create/edit modals.
- **Error handling**: `src/helper/error.ts` — `errorHandler(promise, {showToast, callback})` for one-off imperative calls outside the mutation-hook pattern (e.g. the auth store's login/logout/refresh); unwraps the Eden `{data, error}` envelope, auto-handles `401 Unauthorized` by clearing the session cookie + toasting + redirecting to `/login`.

## Real-time features (WebSocket / Waycast)

Transport: **socket.io-client** (`src/lib/ws.ts`) connecting to the manager over the `/ws` path (proxied in dev via `vite.config.ts`), using `socket.io-msgpack-parser`, and `waycast`'s typesafe client layered on top. Auth is via the `session` cookie read through `cookieStore` and sent in the socket `auth` handshake payload. The socket auto connects/disconnects based on `useAuthStore`'s `user` field.

Full route catalog: [protocols/websocket-waycast.md](../protocols/websocket-waycast.md). Frontend consumption via **`src/hooks/ws.ts`**:

- `useWSConnectionState()` — boolean connected state.
- `useWSData(name, {params, initialData})` — subscribes to a Waycast **data** route, returns the latest value as local state (e.g. `node:[id]:health`).
- `useWSEvent(name, {params, handler})` — same subscription, side-effect only — the dominant pattern, almost always used to patch the TanStack Query cache via `endpoint.setQueryData(queryClient, updater)` instead of maintaining separate local state, so REST-fetched data and websocket pushes share one source of truth.
- `useWSAction(name)` — wraps a Waycast **rpc** call with `send({data, params, onError, onResponse, callbacks})`, auto-disposing the previous call.

**Realtime feature implementations:**

- **Instructor "enrollments" tab** — live student progress roster: listens to `lab:[labId]:enrollment:new|update|remove` and patches the `api.lab({labId}).enrollment.get` query cache directly, CSV export button.
- **Admin dashboard** — live worker fleet monitor: listens to `admin:worker:new|status|metrics`, patches `api.dashboard.admin.get` cache, feeds the rolling `metrics-history-store` for a Recharts history chart; radial stat cards (CPU/Memory/Storage) and a TanStack Table worker list with color-coded usage.
- **Live lab session** — `lab-session:[sessionId]:client-change` (detects another browser tab/device taking over the same session), `lab-session:[sessionId]:ended` (server force-ends session → navigate away + invalidate caches), `lab-session:[sessionId]:checks` (push lab-check completion updates into `lab-checks-session-store`).
- **Node console window** — `node:[id]:health` drives which UI state to show (starting/healthy/unhealthy/died/destroyed) before/while establishing the Guacamole connection; also listens for session client-change to auto-close the popup window if superseded.
- **Device-template "Test Connection"** — uses `useWSAction("device-template:test")` for a request/stream-reply RPC (info/warn/stats logs while the manager spins up a throwaway container to validate the template).

## UI component organization

- **`src/components/ui/`** — shadcn/ui primitives (style `radix-vega`, base color `neutral`), generated/customized via `components.json`. Includes button, dialog, alert-dialog, sheet, sidebar (full app shell primitive), table, tabs, chart (Recharts wrapper), combobox/command (cmdk), calendar/date pickers, resizable (react-resizable-panels), pagination, breadcrumb, field/input-group, empty (empty-state), skeleton/spinner, tooltip, etc.
- **`src/components/forms/`** vs **`src/components/input/`** — two parallel sets: `forms/*-field.tsx` are TanStack-Form-bound field components (registered in `hooks/form/use-app-form.ts`'s `fieldComponents` map), while `input/*-input.tsx` are the underlying plain controlled components reused outside forms too (e.g. table filters).
- **`src/components/data-table/`** — generic wrapper around `@tanstack/react-table`: `DataTable`, `data-table-toolbar`, `data-table-pagination`, `data-table-column-header`, `data-table-combobox-filter`; paired with `use-api-pagination` and per-route `columns.tsx` files. The standard admin/instructor CRUD listing UI.
- **`src/components/infinite-list/`** — parallel generic component for infinite-scroll listings, paired with `use-api-infinite-list`.
- **`src/components/mdx-plugins/lab-checks/`** — custom `@mdxeditor/editor` plugin: lets instructors embed interactive "lab check" blocks inside markdown lab instructions (`edit/` = authoring UI; `view/` = read-only student rendering reflecting live completion state).
- **`shared/topology/components/`** — the network-diagram editor/canvas: `canvas.tsx` (pan/zoom/selection root), `background.tsx`, `layers/` (device/edge/group/note/selection layers, z-ordered), `nodes/` (device, edge, group, note, pending-edge render components), `device-palette.tsx` (drag-source of device templates), `properties/` (side-panel property editors), `toolbar.tsx` + `buttons/`, `modals/interface-select-modal.tsx`. Backed by the slice-based Zustand store and a large set of interaction hooks under `hooks/helper/` (hotkeys, wheel/zoom, drag/pan, drop-from-palette, node select, note editing, screen↔world coordinate conversion).
- **`shared/guacamole/components/`** — `guacamole-connection.tsx` (mounts the Guacamole display + wires mouse/keyboard/clipboard/resize hooks) and `guacamole-connection-states.tsx` (connecting/waiting/error overlays), used exclusively by `routes/lab/.../node/$nodeId.tsx`.
- **`routes/_dashboard/-module/components/`** — shared dashboard chrome: `app-sidebar.tsx` (role-aware nav, built from `constants/menu.ts`), `app-breadcrumb.tsx`, `app-logo.tsx`, `nav-main.tsx`, `nav-user.tsx`, plus reusable `cards/` (stat-card, radial-stat-card, lab-details-card, lab-attachments-card).

## Auth flow (frontend)

1. **Bootstrap**: `__root.tsx`'s `beforeLoad` runs on first navigation; if `useAuthStore.user === undefined` it calls `actions.refresh()`.
2. **`refresh()`**: checks for a `session` cookie via the native `cookieStore` API — if absent, immediately sets `user: null` without hitting the network; if present, calls `api.auth.me.get()` and on success sets `user: data`.
3. **Login**: `/login` route (guarded by `guestRoute()`). Form built with `useAppForm` + typebox `Compile(LoginRequest)` validator. On submit, `useAuthStore.login(credentials)` → `api.auth.login.post(credentials)` → on success calls `refresh()` again. There is also a "Login with CAS" button, a plain `<a href="/api/auth/cas">` — a full page redirect into the manager's server-side CAS/SSO flow (not an SPA XHR).
4. **Route protection**: three guard factories in `lib/middlewares.ts`, all reading `useAuthStore.getState()` synchronously inside a route's `beforeLoad`.
5. **Redirect-after-auth UX**: `__root.tsx`'s `RouteComponent` watches `useAuthStore` and `useRouterState` together — computes a `redirectUrl` and renders `<Navigate replace/>` if login state and current page are out of sync.
6. **Logout**: `useAuthStore.logout()` → `api.auth.logout.post()` → deletes the `session` cookie, `queryClient.clear()`, sets `user: null`.
7. **Websocket auth**: piggybacks on the same `session` cookie — `lib/ws.ts`'s socket.io `auth` callback reads `cookieStore.get("session")` and sends it in the handshake.
8. **401 handling in imperative calls**: `helper/error.ts`'s `errorHandler` specifically detects `401 Unauthorized` responses, clears the session cookie, toasts "Session expired", and redirects to `/login`.

## Notable configuration

**`vite.config.ts`**:

- Plugins: `@tanstack/devtools-vite`'s `devtools()`, `vite-tsconfig-paths`, `@tailwindcss/vite` (Tailwind v4, no separate PostCSS config), `@tanstack/router-plugin`'s `tanstackRouter({ target: "react", autoCodeSplitting: true })`, `@vitejs/plugin-react`.
- `esbuild.drop`: strips `console`/`debugger` statements in production builds only.
- `build.outDir`: `../../out/manager/public` (`emptyOutDir: true`) — **the web app builds directly into the manager app's static-file-serving directory**, i.e. the manager Elysia server serves the built SPA as its frontend (single-deploy-unit architecture, see [overview.md](overview.md)).
- Dev server proxy: `/ws` → `http://localhost:3000` (websocket upgrade), `/display` → `http://localhost:8080` (websocket upgrade — Guacamole's display service, proxied separately from the manager), `/api` → `http://localhost:3000`. No `.env` files are used in this app; all backend URLs are same-origin in production.

**`tsconfig.json`** path aliases: `@web/*` → `./src/*`; `@manager/*` → `../manager/src/*` (type-only cross-app import, used solely for `import type { App } from "@manager/server"`). `moduleResolution: "bundler"`, `verbatimModuleSyntax: true`, strict mode.

**`components.json`** (shadcn CLI config): style `radix-vega`, base color `neutral`, CSS file `src/styles.css`, no `tailwind.config` (v4 CSS-first config), icon library `lucide`.

**Workspace packages this app depends on**: `@jawit/query`, `@jawit/zustand-helper`, `@jawit/common`, `@jawit/paginator`, `@vlab/shared`, `@vlab/ws` — see [shared-packages.md](shared-packages.md).

**Testing**: `vitest` + `@testing-library/react` + `jsdom` (`bun run test`), `tsc --noEmit` for typecheck, Biome for lint/format (root-level, not per-app).
