3. Mekanisme Alat

Arsitektur Manager–Worker. Manager: REST API, WebSocket, guacamole-lite, BullMQ. Worker (1…n): containerlab + Docker + guacd + Evaluator.

1
Instruktur susun topologi drag-and-drop + instruksi markdown + kriteria penilaian dari katalog
2
Peserta mulai sesi — vLab deploy container MikroTik CHR & Linux ke Worker node via Containerlab
3
Akses CLI/GUI langsung dari browser via Apache Guacamole (SSH, RDP, VNC)
4
Evaluator deteksi perubahan konfigurasi via RouterOS API /listen & ip monitor — bukan polling
5
Status lulus/gagal per kriteria tampil real-time ke peserta; skor dihitung otomatis saat sesi berakhir

Peserta mengakses terminal MikroTik RouterOS (kiri) dan desktop Ubuntu Linux (kanan) secara bersamaan — semuanya dari tab browser.

5. Kesimpulan & Saran

vLab terbukti mampu menjalankan lab jaringan yang realistis, dinilai otomatis, dan skalabel — relevan dengan kebutuhan pasar MikroTik di Indonesia.
