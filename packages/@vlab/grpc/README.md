# @vlab/grpc

## Overview

gRPC definitions and `waycast` router for communication between the vLab Manager and Worker nodes. It encapsulates the gRPC API using `nice-grpc` and provides typed schemas and command routes using `waycast` and `@sinclair/typebox`.

## Installation

This package is an internal workspace package. Add it as a workspace dependency:

```json
{
  "dependencies": {
    "@vlab/grpc": "workspace:*"
  }
}
```

## Key Features

- **Worker Management**: Routes for deploying and destroying containerlab topologies.
- **Evaluation**: Routes for starting and stopping real-time evaluation sessions on nodes.
- **Monitoring**: Real-time event streaming for node health and interface updates.
- **Service Bindings**: Wraps `nice-grpc` autogenerated client/server interfaces for direct TS usage.

## Usage & API Examples

```ts
import { AppRouter } from "@vlab/grpc";

// The waycast AppRouter exposes modular RPC and data routes:
const client = AppRouter.createClient({
  endpoint: "localhost:50051",
});

// Call a gRPC route through the waycast router
const result = await client.rpc("clab:deployLab", {
  id: "session-123",
  topology: { /* ... */ },
});
```

## Development & Scripts

Commands:

| Script | Description |
|---|---|
| `bun run build` | Build protocol buffers into TypeScript schemas (`grpc_tools_node_protoc` & `ts-proto`) |
| `bun run check` | Run linter and formatting checks globally |
| `bun run typecheck` | Run TypeScript type checks globally |
